[SLIDE 9 - CHALLENGES & SOLUTIONS]

**Challenges faced aur unka solution:**

**Challenge 1: Multiple AI Models Integration**
- Problem: Different models ki different APIs
- Solution: Config file mein centralized model management

**Challenge 2: Image Generation Async API**
- Problem: Image instantly generate nahi hoti
- Solution: Polling mechanism implement kiya (2 sec interval)

**Challenge 3: UTF-8 Encoding Issues**
- Problem: Emojis aur special characters corrupt ho rahe the
- Solution: Proper encoding headers aur ensure_ascii=False

**Challenge 4: Chat History Management**
- Problem: Database nahi chahiye tha
- Solution: JSON file-based storage with UTF-8 encoding

**Challenge 5: Real-time Streaming**
- Problem: Buffering issues
- Solution: Server-Sent Events with proper chunking

## 📝 Q&A PREPARATION (Expected Questions)

```
Q1: Aapne Flask kyun choose kiya?
A: Flask lightweight hai, easy to learn, aur humari requirements ke liye perfect tha. Django bahut heavy hota complex projects ke liye.

Q2: Free AI models reliable hain?
A: Haan, OpenRouter ke through hum multiple free models access kar rahe hain jo production-grade hain. Aur humne different models different tasks ke liye use kiye performance ke liye.

Q3: Chat history database mein kyun nahi?
A: Initial version mein simplicity ke liye JSON file use ki. Future mein database integrate karenge scalability ke liye.

Q4: Security kaise handle ki?
A: API keys config file mein, input validation, file upload restrictions, aur UTF-8 encoding properly implement ki hai.

Q5: Image generation slow kyun hai?
A: Kyunki asynchronous API hai - image generate hone mein time lagta hai. Humne polling mechanism diya hai jo har 2 seconds mein check karta hai.

Q6: NumGo API legal hai?
A: Haan, yeh educational purposes ke liye professors ne provide ki hai. Legitimate phone number lookup service hai.

Q7: Mobile par kaisa work karta hai?
A: Fully responsive design hai. Mobile par sidebar collapsible hai, touch-friendly buttons hain.

Q8: Kitne concurrent users handle kar sakta hai?
A: Current setup mein 10-20 simultaneous users. Production ke liye gunicorn aur nginx use karenge.
